# 变电站智能监测与维保管理系统需求分析

## 1. 项目背景与核心痛点

变电站运行过程中，变压器、母线、开关柜等关键设备需要长期监测电流、温度、电压等指标。传统固定频率采样存在明显矛盾：

- 如果长期按秒级采样，故障追忆数据足够细，但 Oracle 21c XE 存储压力大，历史数据膨胀快。
- 如果长期按分钟级采样，存储成本可控，但故障发生瞬间的细节不足，难以还原事故过程。

PSM-Smart 的核心目标是通过自适应动态采样解决该矛盾：设备平稳时按 1 分钟 1 次保存常规数据；当电流、温度或温升变化率超过阈值时，自动进入 1 秒 1 次的高频录波模式，并在告警前后提供事故追忆曲线。

## 2. 用户角色

### 系统管理员

负责维护系统基础配置，包括变电站、间隔、设备、测点、阈值、用户权限和系统参数。

### 运行监控人员

负责查看实时监测工作台，关注设备状态、趋势曲线、告警事件和事故追忆数据。

### 维保工程师

负责接收告警自动生成的维保工单，处理现场问题，填写处理反馈并完成归档。

### 管理人员

负责查看告警统计、工单处理状态、设备健康趋势和运维闭环效率。

## 3. 功能需求

### 3.1 资产台账管理

系统采用四级资产模型：

```text
Station -> Bay -> Device -> Tag
```

功能要求：

- 支持变电站基础信息维护，包括名称、电压等级、位置。
- 支持间隔信息维护，并归属到具体变电站。
- 支持设备信息维护，并归属到具体间隔。
- 支持测点信息维护，并归属到具体设备。
- 设备和测点应能作为采样、告警、曲线查询和工单追溯的基础对象。

### 3.2 阈值配置

设备或测点需要支持独立阈值：

- 越限阈值：例如负荷电流上限、油温上限、绕组温度上限。
- 变化率阈值：例如温升变化率阈值。
- 采样策略参数：常规采样周期、突发采样周期、突发保持时间。

阈值配置用于动态采样状态切换、告警生成和维保派单。

### 3.3 动态采样

系统包含两种采样模式：

- Normal 模式：设备平稳时按 1 分钟 1 次写入采样数据，`FREQ_FLAG = 0`。
- Burst 模式：检测到越限或变化率异常时按 1 秒 1 次写入采样数据，`FREQ_FLAG = 1`。

切换规则：

- 当电流、温度或温升变化率超过阈值时，从 Normal 切换到 Burst。
- 进入 Burst 后至少持续 5 分钟。
- Burst 持续期间如果数据仍异常，则延长高频录波。
- Burst 持续期结束且数据恢复平稳后，回到 Normal。

### 3.4 告警追忆

系统应支持按告警事件下钻到高频数据：

- 告警列表展示设备、测点、告警类型、告警等级、开始时间、结束时间和处理状态。
- 点击告警后查询告警开始时间前 5 分钟到后 5 分钟的数据。
- 事故追忆曲线优先展示 `FREQ_FLAG = 1` 的秒级数据。
- 如果高频数据不足，可同时展示分钟级数据作为背景趋势。

### 3.5 维保闭环

严重告警触发后，系统自动生成维保工单：

```text
触发告警 -> 高频录波 -> 生成工单 -> 工程师处理 -> 填写反馈 -> 流程归档
```

工单状态：

- `0`：待办。
- `1`：处理中。
- `2`：已完成。

功能要求：

- 严重告警自动写入 `MAINT_TASK`。
- 维保工程师可更新工单状态。
- 维保工程师可填写反馈说明。
- 前端应提供独立“工单管理”界面，支持按工单状态、设备、时间和关键词查询。
- 工单完成后保留告警和设备追溯关系。

### 3.6 查询与运行日志

消息查询界面应提供设备状态总览，并支持点击设备总数、正常设备、告警设备、停运设备卡片查看对应设备列表。告警设备数量非 0 时使用红色强调，停运设备数量非 0 时使用黄色强调。

历史数据查询默认使用当前系统时间作为结束时间并向前查询 1 小时。用户可手动指定开始时间、结束时间和采样频率，不再提供“最近已有数据”入口。

历史数据列表应按采样时间聚合展示：同一时间点采集的多个测点只显示在一行内，并给出整体状态。整体状态按照明细中的最高风险质量标记计算，优先级为无效、可疑、正常。用户可点击某一采样时间行查看该时间点的设备、测点、数值、频率和质量明细。

系统管理员登录后应能访问“运行日志”界面，合并查看前端操作/API 日志和后端 API 请求/异常日志。运行日志默认展示 `INFO` 及以上级别，并支持按 `DEBUG`、`INFO`、`WARN`、`ERROR` 过滤。

### 3.7 模拟采样与合理范围

ADMIN 模拟功能应满足以下要求：

- 正常模式按 1 分钟周期写入 `FREQ_FLAG = 0` 的采样数据。
- 异常模式按 1 秒周期写入 `FREQ_FLAG = 1` 的高频采样数据。
- 前端“写入采样”“生成告警”“生成工单”“最近写入”应在模拟运行期间自动刷新。
- 传感器数值应按测点类型限定演示合理范围。负荷电流以测点额定告警阈值为基准，正常值不超过阈值，异常值超过阈值；油温和绕组温度参考电力变压器负载导则中顶层油温、热点温度限值的常见工程区间；母线电压参考供电电压允许偏差的常见范围；环境温度按站内环境传感器演示范围限定。
- 超出合理范围的采样应标记为可疑质量，并生成严重告警和维保工单。

资料依据：

- IEC 60076-7:2018 适用于矿物油浸式变压器，给出负载、运行温度和热老化方面的导则：https://webstore.iec.ch/en/publication/34351
- EN 50160:2022 规定公共电网供电电压特性的正常运行范围：https://www.intertekinform.com/en-us/standards/en-50160-2022-353902_saig_cenelec_cenelec_3209237/

当前演示实现采用工程化简化范围，不替代现场保护定值或正式运维规程。

## 4. 非功能需求

### 4.1 Oracle 21c 兼容

- 主键继续使用 `SEQUENCE`，与 MyBatis 批量写入和显式取号策略保持一致。
- SQL 以 Oracle 21c XE 为目标环境，避免使用会影响当前演示脚本幂等性的语法。
- 采样时间使用 `TIMESTAMP`，满足秒级数据记录需求。
- 当前初始化脚本使用非分区表，优先保证 Oracle 21c XE 本地环境可执行。

### 4.2 批量写入能力

- 高频采样数据先进入 Java 端 `BlockingQueue`。
- 后台消费者使用 MyBatis `ExecutorType.BATCH` 批量写入 Oracle。
- 避免秒级采样逐条同步提交导致数据库 I/O 压力过大。

### 4.3 查询性能

- `TS_RAW_DATA` 必须建立 `(DEVICE_ID, SAMPLE_TIME)` 复合索引。
- 高频追忆按设备和时间范围查询。
- 告警列表按设备和开始时间查询。
- 工单按告警 ID 和状态查询。

### 4.4 可演示性

- 初始化 SQL 应包含完整演示闭环数据。
- 演示数据需要同时包含 `FREQ_FLAG = 0` 和 `FREQ_FLAG = 1`。
- 演示数据需要包含告警和工单，便于前端直接展示。

### 4.5 前后端分离

- 后端提供 REST API。
- 前端通过 API 获取台账、监测值、曲线、告警和工单。
- 前端使用 Vue 3、TypeScript、Element Plus 和 ECharts 实现监测工作台。

### 4.6 可观测性

- 后端应记录 `/api/**` 请求路径、状态码、耗时和异常摘要。
- 前端应记录接口调用成功、失败以及关键页面切换操作。
- 当前课设演示版本运行日志以内存和浏览器本地存储保留最近记录，不强制落库。

## 5. 数据需求

### 5.1 核心表

| 表名 | 作用 | 关键字段 |
| :--- | :--- | :--- |
| `STATION_BASE` | 变电站基础信息 | `ID`, `NAME`, `VOLTAGE_LEVEL`, `LOCATION` |
| `BAY_BASE` | 间隔信息 | `ID`, `STATION_ID`, `NAME`, `BAY_TYPE` |
| `DEVICE_BASE` | 设备信息 | `ID`, `BAY_ID`, `NAME`, `DEVICE_TYPE`, `CURRENT_LIMIT`, `TEMP_LIMIT`, `TEMP_RATE_LIMIT` |
| `TAG_BASE` | 测点信息 | `ID`, `DEVICE_ID`, `TAG_CODE`, `TAG_NAME`, `UNIT`, `WARN_LIMIT`, `RATE_LIMIT` |
| `TS_RAW_DATA` | 采样数据 | `ID`, `DEVICE_ID`, `TAG_ID`, `SAMPLE_TIME`, `VAL`, `FREQ_FLAG` |
| `ALARM_LOG` | 告警记录 | `ID`, `DEVICE_ID`, `TAG_ID`, `ALARM_TYPE`, `ALARM_LEVEL`, `START_TIME`, `STATUS` |
| `MAINT_TASK` | 维保工单 | `TASK_ID`, `ALARM_ID`, `STATUS`, `ASSIGNEE`, `FEEDBACK` |

### 5.2 主外键关系

- `BAY_BASE.STATION_ID` -> `STATION_BASE.ID`
- `DEVICE_BASE.BAY_ID` -> `BAY_BASE.ID`
- `TAG_BASE.DEVICE_ID` -> `DEVICE_BASE.ID`
- `TS_RAW_DATA.DEVICE_ID` -> `DEVICE_BASE.ID`
- `TS_RAW_DATA.TAG_ID` -> `TAG_BASE.ID`
- `ALARM_LOG.DEVICE_ID` -> `DEVICE_BASE.ID`
- `ALARM_LOG.TAG_ID` -> `TAG_BASE.ID`
- `MAINT_TASK.ALARM_ID` -> `ALARM_LOG.ID`

### 5.3 索引策略

- `IDX_TS_RAW_DEVICE_TIME`：优化设备曲线和事故追忆查询。
- `IDX_TS_RAW_TAG_TIME`：优化测点级曲线查询。
- `IDX_ALARM_DEVICE_TIME`：优化设备告警列表。
- `IDX_TASK_ALARM`：优化告警到工单的关联查询。

## 6. 业务流程

### 6.1 正常监测流程

```text
读取台账与阈值
  -> 定时采集设备数据
  -> 判断数据平稳
  -> 写入分钟级数据
  -> 前端展示日/周趋势曲线
```

### 6.2 异常告警流程

```text
采集当前数据
  -> 计算越限和变化率
  -> 触发告警
  -> 切换 Burst 高频采样
  -> 写入秒级数据
  -> 严重告警生成工单
```

### 6.3 事故追忆流程

```text
用户点击告警
  -> 查询告警前后 5 分钟数据
  -> 优先读取秒级高频数据
  -> ECharts 展示高频波形
  -> 辅助工程师判断故障过程
```

### 6.4 维保归档流程

```text
工程师接收工单
  -> 状态改为处理中
  -> 现场处理
  -> 填写反馈
  -> 状态改为已完成
  -> 告警与工单归档可追溯
```

## 7. 验收标准

### 7.1 数据库验收

- Oracle 21c 初始化脚本可执行。
- 7 张核心表均创建成功。
- 7 个序列均创建成功。
- `TS_RAW_DATA` 包含 `FREQ_FLAG` 字段。
- `TS_RAW_DATA` 包含 `FREQ_FLAG = 0` 和 `FREQ_FLAG = 1` 的演示数据。
- `ALARM_LOG` 和 `MAINT_TASK` 能通过外键关联。

### 7.2 后端验收

- Spring Boot 能连接 Oracle 21c。
- 能查询资产台账。
- 能写入分钟级采样数据。
- 能在异常波动时按 1 秒周期写入秒级高频数据。
- 严重告警能生成告警记录和维保工单。
- 事故追忆接口能返回告警前后 5 分钟数据。
- 历史数据在未指定时间范围时按当前时间最近 1 小时查询。
- 采样值超出合理范围时能生成严重告警和维保工单。
- 工单查询和状态反馈接口可用。
- ADMIN 可通过 `/api/runtime-logs` 查询后端运行日志。

### 7.3 前端验收

- 首屏为监测工作台。
- 能展示站点、间隔、设备、测点层级。
- 能展示设备最新状态和分钟级趋势曲线。
- 能展示告警列表和维保工单列表。
- 历史数据列表按同一采样时间聚合，可点击查看明细。
- 点击告警能查看高频事故追忆曲线。
- 点击设备状态卡片能查看对应设备列表。
- 能通过工单管理界面更新状态和反馈。
- ADMIN 模拟测试指标能实时刷新。
- ADMIN 可查看前后端运行日志，默认级别为 `INFO`。

### 7.4 业务闭环验收

- 初始化台账可用。
- 正常数据可展示。
- 异常数据可触发告警。
- Burst 高频数据可追忆。
- 工单可从告警自动生成。
- 工单可处理、反馈、归档。
